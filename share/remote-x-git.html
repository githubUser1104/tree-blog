    # 基础用法

    • 问题：

      • WINDOWS下会出现LF will be replaced by CRLF 的 警告：配置git config --global core.autocrlf true，
      • 在git项目中出现了嵌套的git项目
        • 方法一：删除内层的.git文件夹
        • 方法二：git的submodule方式管理

    • 调出命令行
      项目根目录右键 git bash here

    • 创建版本库
      git init

    • 添加文件
      新建readme.md文件
      git add readme.md（批量添加：git add .）

    • 删除文件
      删除目标文件
      git add .

    • 提交文件（增删改后都要提交）
      git commit -m "message"

    • 查看仓库当前状态
      git status

    • 查看文件与仓库版本的差异
      git diff readme.md

    • 查看仓库日志
      git log（单行显示：git log --pretty=oneline）

    • 版本回退
      git reset --hard HEAD~n（n表示要回退的次数，取1就是回退到上一个版本）

    • 版本跳转
      git reset --hard 版本号（版本号可以通过git reflog查看）

    # 分支管理

    • 新建分支
      git branch dev

    • 查看当前分支（默认分支为master）
      git branch

    • 跳转到dev分支
      git checkout dev

    • 跳转回主分支
      git checkout master

    • 将dev分支的修改合并到当前分支（master）
      git merge dev

    • 删除分支
      git branch -d dev

    # 远程仓库

    github

    • 获取SSH加密传输密钥（选项使用默认值即可）
      ssh-keygen -t rsa -C "描述"

    • 绑定密钥到github（允许添加多个Key，只要把每台电脑的Key都添加到GitHub）
      mac：将/Users/zeng/.ssh/id_rsa.pub的内容复制到gitub账户的SSH Key win：在C:\Users\jack.ssh找到上述命令生成id_rsa.pub公钥，复制里面的内容

    • 登陆GitHub，打开“Account settings”，“SSH Keys”页面，点“Add SSH Key”，填上任意Title ，在Key文本框里粘贴id_rsa.pub文件的内容，点“Add Key”

    • 先有本地库，再关联远程库
      git remote add origin git@github.com:GITHUB账号名/GITHUB仓库项目名.git

    • 提交到远程库 git push -u origin master（第一次带上-u参数将本地master与远程master连接起来，如果本地分支为main就git push -u origin main）
      git push origin master（第二次以及之后可以不带-u，还可以简写为git push）
      git push origin dev（如果没有该分支会自动创建）

    • 先有远程库，再克隆到本地库，如果clone过程中有ssh key提示确认，需要手动输入 yes 而不能直接回车

    git clone git@github.com:GITHUB账号名/GITHUB仓库项目名.git
git add .
git commit -m "update"
git push

    • 抓取远程库

    • 查看远程库地址 git remote -v

    • 修改远程库地址（先删除原本的再增加新的） git remote rm origin git remote add origin git@github.com:GITHUB账号名/GITHUB仓库项目名.git

    • 冲突 本地库的文件A修改了但未提交到远程库，同时其他人把远程库的文件A修改了，然后提交本地库的文件A就会发生冲突
      解决：在提交之前先pull最新版的远程库，如果有冲突就合并冲突的代码，然后再提交。

    • 只下载某个文件夹 网页上只能下载整个下项目，如果只想要下载某个文件夹，可以

    # fork远程库

    # 设他人的远程库A，被fork到自己的远程库B
# 在建立B的本地库
git clone B的地址
# 本地库关联远程库B和A
git remote add origin B的地址
git remote add upstream A的地址
# 根据A库的master分支，创建本地库的myMaster分支，并跳转到该分支
git checkout -b myMaster --track upstream/master
# 修改本地库的myMaster，提交到远程库B
git add .
git commit -m ""
git push -u origin myMaster
# 在页面发起merge request，将远程库B的修改 同步 到远程库A

# 当远程库A被他人更新了，需要更新远程库
git fetch upstream
# 将远程库A的master代码同步到本地的myMaster分支
git pull upstream master:myMaster

# 参考
git push origin <本地分支名>:<远程分支名>
git pull origin <远程分支名>:<本地分支名>
git push -u origin <本地分支名> # 将本地分支与远程同名分支相关联，后序可以简写git push 和 git pull
